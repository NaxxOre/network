Topology showcase (for school demo)

Goal: show end-to-end monitoring from network traffic to app-level threat detection.

1) Simple topology
- Node A: victim host running Docker stack (WAC + OpenSearch + dashboards)
- Node B: normal user client
- Node C: attacker client (failed logins / brute force / many sessions)
- Optional router/firewall node between clients and Node A

2) Demo flow
- Normal login from Node B -> success event in dashboards + Telegram
- Failed repeated login from Node C -> brute-force signal
- Login using multiple browsers/devices with same account -> parallel session violation
- Add X-Country header changes in API calls -> geography anomaly

3) Minimum evidence to present
- Screenshot: topology map
- Screenshot: dashboards panels changing in real time
- Screenshot: Telegram alerts with request_id, source_ip, session_id, device
- Correlation table: time, source IP, account, threat_type, action

Important note about client IP on Windows Docker Desktop

- Seeing source_ip=172.19.0.1 can be normal on Docker Desktop because of NAT between host and Linux VM.
- If you need true per-device LAN IP in logs for a graded demo, run WAC in a Linux VM/host (or in GNS3) with bridged networking.

Threat-pattern demo commands (quick)

1) Account brute force (same account, many failed attempts)
- CMD:
for /L %i in (1,1,7) do curl -s -o nul -w "code=%{http_code}\n" -X POST http://localhost:8000/login -d "username=test&password=Wrong123!"
- PowerShell:
1..7 | ForEach-Object { curl.exe -s -o NUL -w "code=%{http_code}`n" -X POST http://localhost:8000/login -d "username=test&password=Wrong123!" }

2) Credential stuffing (one source tries many usernames)
- CMD:
for %u in (alice bob carol dave erin frank) do curl -s -o nul -w "%u code=%{http_code}\n" -X POST http://localhost:8000/login -d "username=%u&password=Wrong123!"
- PowerShell:
"alice","bob","carol","dave","erin","frank" | ForEach-Object { curl.exe -s -o NUL -w "$_ code=%{http_code}`n" -X POST http://localhost:8000/login -d "username=$_&password=Wrong123!" }

3) Distributed account attack (same account from many sessions)
- CMD:
for /L %i in (1,1,4) do curl -s -o nul -w "sess-%i code=%{http_code}\n" -X POST http://localhost:8000/login -H "X-Session-ID: attacker-%i" -d "username=test&password=Wrong123!"
- PowerShell:
1..4 | ForEach-Object { curl.exe -s -o NUL -w "sess-$_ code=%{http_code}`n" -X POST http://localhost:8000/login -H "X-Session-ID: attacker-$_" -d "username=test&password=Wrong123!" }

Geo/VPN quick test

1) Trigger geo anomaly for same user (US then DE):
curl -i -H "X-WebID: https://alice.example/profile#me" -H "X-Country: US" -H "X-Session-ID: geo-1" http://localhost:8000/r/docs/report.txt
curl -i -H "X-WebID: https://alice.example/profile#me" -H "X-Country: DE" -H "X-Session-ID: geo-2" http://localhost:8000/r/docs/report.txt

2) Force VPN flag for demo:
curl -i -H "X-WebID: https://alice.example/profile#me" -H "X-Country: US" -H "X-VPN-Suspected: true" -H "X-Session-ID: vpn-1" http://localhost:8000/r/docs/report.txt

Expected threat types:
- vpn_geography_anomaly
- vpn_proxy_network

One-day SOC showcase flow (recommended)

Fast Demo Start

1) Start core stack:
- cd WAC
- docker compose up -d opensearch resource dashboards nginx
- docker compose stop guard

2) Start host guard (real LAN source IP + Telegram from host runtime):
- .\scripts\run-host-guard.ps1 -ForceFreePort

If you see port error [Errno 10048] on :8000:
- Get-NetTCPConnection -LocalPort 8000 -State Listen | Select-Object OwningProcess
- Stop-Process -Id <PID> -Force
- Re-run: .\scripts\run-host-guard.ps1 -ForceFreePort

If you see 502 Bad Gateway on http://localhost/dashboards/:
- docker compose up -d opensearch dashboards nginx
- wait 15-30 seconds, then open http://localhost/dashboards/

3) Upgrade dashboard export with showcase panels:
- powershell -ExecutionPolicy Bypass -File .\scripts\upgrade_showcase_ndjson.ps1
- Import NDJSON in Dashboards (Saved Objects -> Import): WAC\opensearch\webauthguard-security-dashboard.ndjson
- (Optional if dashboard already imported): powershell -ExecutionPolicy Bypass -File .\scripts\update_showcase_dashboard.ps1
